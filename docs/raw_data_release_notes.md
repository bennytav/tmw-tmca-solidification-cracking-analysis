# Raw-data release notes

The repository includes raw controller XML files so that readers can practice
the full workflow and reproduce the manuscript-level analysis.

## Recommended public layout

```text
raw_data/TMCA/
raw_data/TMW/
data/manifest/benny_tmca_manifest_prefilled.csv
data/manifest/benny_tmw_manifest_prefilled.csv
```

The raw XML files are the source controller outputs.  The standardized CSV files
are generated by the code and should not be edited manually.

## GitHub size note

The uncompressed raw data are large because XML controller logs are text files.
No single file in the provided archive is above 100 MB, but the repository is
large.  If GitHub upload or cloning becomes slow, keep the code on GitHub and
upload the raw-data archive to Zenodo, then link the Zenodo DOI from the
GitHub README and the manuscript.

## What not to change

Do not edit the raw XML files.  If a run needs correction, edit the manifest:

- `weld_start_time_s`
- `weld_start_method`
- `l_surf_mm`, `l_ct_mm`, `l_crack_mm`, or `l_star`
- `include_in_summary`
- `notes`

This keeps the raw data immutable and the analysis decisions traceable.
